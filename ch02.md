## 2-1 Codex 開發基本技巧
### 使用 /init 建立 AGENTS.md 設計專案規範

```md
# 專案規範

## 使用 uv 管理 Python 環境

專案中需要使用 Python 時：

- 一律使用 uv 建置 Python 環境，不要使用其他工具
- 一律將 Python 安裝到 uv 的全域環境中，不要裝到專案內
- 若未指明，請使用 Python 3.13
- 必要時使用 `uv init` 初始化專案
- 使用 `uv add/remove` 管理套件
- 使用 `uv run` 執行 Python 腳本檔
```

### 用 /plan 命令規劃 MVP

```
我要用 Python 撰寫一個永遠在最上層顯示 Codex 剩餘用量的小工具，請參考 https://developers.openai.com/codex/app-server 文件，使用 account/rateLimits/read API 取得剩餘用量資訊
```

## 2-2 使用工作樹開發新功能
### 用工作樹為專案建立新聊天室
### 撰寫新功能的產品需求書

```md
我要加上以下功能：

# 剩餘用量面板工具進化版產品需求書

1. 增加多國語言版本
    - 提供繁體中文與英文語系
    - 預設採用系統語系，若非繁體中文或英文，採用英文
2. 增加淺色/深色主題功能
    - 提供以 Tiffany Blue 配色的淺色主題與 Monokai 配色的深色主題
    - 預設採用系統配色主題
3. 面板提供透明效果，0% 為完全透明，100% 為完全不透明，預設為 75% 透明度
4. 前兩項功能，要在面板最上方提供以圖示表示的按鈕操控，透明程度使用搖桿控制，都採用現代扁平風格，不要傳統 Tkinter 那種立體死硬風格
```
