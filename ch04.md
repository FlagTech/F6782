# 第 4 章 幫 Codex 接上各種超能力

## 4-1 讓 Codex 透過內建瀏覽器看見網頁結果

### 下載並啟動「密碼管理助手」（PassNest）專案

- [PassNest GitHub 網址](https://github.com/FlagTech/PassNest)

- 複製專案：

```md
git clone https://github.com/FlagTech/PassNest
```

- 設定環境並啟動應用程式：

```md
cd PassNest
npm install
npm run dev
```

### 網頁應用程式自動化測試

- 提示內容：

```md!
@瀏覽器
請進入到 http://localhost:5173 測試網頁應用程式的功能是否正常。

先以一般桌面尺寸確認頁面標題、側邊欄、搜尋欄與「新增密碼」按鈕都能顯示，
並檢查 Console 是否出現錯誤。

接著切換成 375 × 812，確認頁面沒有水平溢出、文字截斷或控制項重疊。
```

- 提示內容：

```md!
請使用響應式設計，幫我修正手機板問題
```

- 提示內容：

```md!
@瀏覽器
請在 http://localhost:5173 執行以下測試：

1. 點擊「新增密碼」。
2. 名稱填入「測試帳號」。
3. 網址填入「https://example.com」。
4. 帳號填入「test@example.com」。
5. 點擊產生密碼按鈕，確認密碼欄位已產生內容，但不要把內容回報給我。
6. 點擊「新增」。
7. 確認清單中出現「測試帳號」，並檢查 Console 是否有錯誤。
```

## 4-3 透過 Skill 打包作業流程


### 用 $skill-creator 建立 web-digest Skill

- 提示內容：

```md!
$skill-creator
請把我們剛剛整理 Gmail 郵件、抓取 HBO Max 片單的流程，整理成一個叫 web-digest 的 Skill，存到 ~/.agents/skills/<技能名稱>/ 底下。

這個 Skill 必須執行以下強制流程：

1. 一律使用 @Chrome 開啟並擷取網站，不得改用其他瀏覽器。
2. 若尚未登入，使用 PassNest，並清楚記載 `/status`、`/entries`、`/autotype`、視窗驗證、MFA 及密碼保護流程。
3. 整理網站內容時，保留標題、日期、來源、摘要、重要性及網址。
4. 每次都必須產生 Excel，不需要使用者另外要求。
5. 一律使用 @Computer Use 開啟 Excel、檢查內容，再透過「另存新檔」儲存到桌面。
6. 完成前確認桌面檔案存在、非空白且可開啟。
7. 支援 Gmail、HBO Max、YouTube、YouTube Music、Notion及其他網站。

請將詳細規則拆分至 references，更新 `agents/openai.yaml`，最後執行 `quick_validate.py`。驗證成功才算完成。
```

### 呼叫並更新 web-digest Skill

- 提示內容：

```md!
$web-digest
登入 YouTube，整理 YouTube Music 中最新發行的音樂資訊
```

- 提示內容：

```md!
$skill-creator
請更新 web-digest SKILL，將資料表儲存在「桌面/網站檔案統整」資料夾中，並且檔名用以下格式：

<網站>_<內容類型>_<YYYY-MM-DD>.xlsx
```
