## 5-1 Subagent 簡介

- 使用 Subagent 並行處理：
    ```md
    請使用多個 subagents 分工審查 task-tracker-demo 專案。

    分工方式：
    1. 一個 subagent 專門重現 README 中的手動操作流程，找出使用者最容易遇到的明顯問題。
    2. 一個 subagent 專門檢查 CLI 操作流程是否直覺，例如選單、輸入提示、錯誤訊息與操作順序。
    3. 一個 subagent 專門檢查資料是否能正確保存與重新載入，確認「儲存並離開」後再次啟動是否符合預期。
    4. 一個 subagent 專門整理可以改善使用體驗與維護性的方向，例如錯誤提示、測試覆蓋、文件說明。
    5. 最後統整所有 subagent 的處理結果，依「對使用者影響程度」順序，簡單列出發現的問題。

    請先不要修改檔案，只輸出審查結果與建議修正方向。
    ```

    5.統整審查結果，依「對使用者的影響程度」排序，簡單列出發現的問題。

- 指定 Subagent 類型進行針對性分工：
    ```md
    請使用 Codex 內建的 explorer 與 worker subagents，檢查並修正 task-tracker-demo 專案。

    第一階段：由 explorer 核對問題並規劃修正任務

    請派出一個 explorer Subagent：
    1.以前面的審查結果為基礎，核對各項問題是否確實存在。
    2.根據問題的影響程度、修改範圍與彼此關係，自行建立適合這個專案的分類方式。
    3.說明每個問題所屬的分類、涉及的檔案、相依關係及建議驗證方式。
    4.只負責檢查與分類，不要修改檔案。

    第二階段：由多個 worker 執行修正

    Explorer 完成後，請根據分類結果安排 worker Subagent：
    1.影響較大、修改風險較高或需要獨立驗證的問題，原則上分別交給不同的 worker 處理。
    2.影響較小且涉及相同功能或檔案的問題，可以合併交給同一個 worker。
    3.根據問題之間的相依關係與檔案衝突，決定哪些工作可以並行，哪些需要依序執行。
    4.worker 的數量由實際分類與分工結果決定。
    5.每個 worker 完成後，回報修改內容與測試結果。

    最後請由主對話整合所有修改、執行完整測試，並回報修正結果。
    ```
    
## 5-2 客製化自己的 Subagent

- 建立專門翻譯技術文件的 Custom Subagent
    ```md
    請參考 Codex 官方 Custom Subagents 文件：https://developers.openai.com/codex/subagents

    請在目前專案建立一個專案用的 Custom Subagent。要求如下：
    1. Subagent 名稱：technical_doc_translator
    2. 用途：將英文技術文件翻譯成自然的臺灣繁體中文
    3. 使用 TOML 格式
    4. 使用 gpt-5.4-mini 模型，推理強度設定為低
    5. Subagent 的實際行為規則：
       - 保留原本的 Markdown、code block 與 inline code 等等的結構
       - 不翻譯 CLI 指令、API 名稱、設定鍵、檔案路徑與 URL
       - 技術術語前後一致
       - 產出的文字風格要清楚、準確，適合臺灣的軟體工程師閱讀，不加入原文沒有的補充說明
       - 如果原文有明顯歧義，保守翻譯並標註不確定處
    6. 建立後請簡短說明這個 Subagent 的用途與使用方式
    ```

## 5-3 讓多個 Subagent 組成專業團隊

- 不使用 Subagent 製作網頁
    ```md
    請搜尋今天臺灣的 10 則重要新聞，參考 https://time.com/ 的首頁風格，製作一個響應式新聞網頁。

    要求：
    1. 包含 1 則頭條新聞與 9 則一般新聞。
    2. 每則新聞要有標題、摘要、來源與原始連結。
    3. 版面要有雜誌式新聞首頁的感覺，不要只是資料清單。
    4. 不要複製 TIME 的商標、圖片或原站內容。
    ```

- 用專精不同工作的 Subagent 協作
  - 建立蒐集臺灣新聞的 Subagent：
    ```md
    請參考 Codex 官方 Custom Subagents 文件：https://developers.openai.com/codex/subagents

    請在目前專案建立一個專案用的 Custom Subagent。

    名稱：taiwan_news_researcher

    主要工作：
    搜尋今天臺灣的 10 則重要新聞，整理每則新聞的標題、來源、日期與原始連結。

    工作要求：
    1. 必須使用可信來源，不要捏造新聞。
    2. 優先使用臺灣主流媒體、官方公告或可信國際媒體。
    3. 如果來源資訊不足，要清楚標註不確定。
    4. 結果要簡潔，方便 parent agent 或其他 Subagent 使用。
    5. 如果需要存檔，只存成精簡的 taiwan_news_sources.md。
    ```

  - 建立研究 TIME.com 風格的 Subagent：
    ```md
    請參考 Codex 官方 Custom Subagents 文件：https://developers.openai.com/codex/subagents

    請在目前專案建立一個專案用的 Custom Subagent。

    名稱：time_style_researcher

    主要工作：
    研究 https://time.com/ 的首頁或新聞頁版面風格，整理成可供網頁製作者使用的設計方向。
    
    工作要求：
    1. 不要寫冗長報告。
    2. 只整理最重要的 5 個視覺規則。
    3. 規則要能直接指導網頁製作，例如標頭、欄位、字體層級、配色與新聞密度。
    4. 可以參考 TIME.com 的設計精神，但不要要求複製商標、圖片或原站內容。
    5. 結果直接回報給 parent agent；除非 parent agent 要求，否則不用另外寫檔。
    ```

  - 建立製作新聞網頁的 Subagent：
    ```md
    請參考 Codex 官方 Custom Subagents 文件：https://developers.openai.com/codex/subagents

    請在目前專案建立一個專案用的 Custom Subagent。

    名稱：news_page_builder

    主要工作：
    根據其他 Subagent 提供的新聞資料與風格規則，製作一個具有雜誌感的臺灣新聞首頁。

    工作要求：
    1.擔任最後整合者，整理新聞的重要性與閱讀順序，而不是直接拼接資料。
    2.從新聞資料中選出 1 則頭條新聞，其餘 9 則作為一般新聞。
    3.每則新聞都要包含標題、精簡摘要、來源與原始連結。
    4.將研究所得的視覺規則實際轉換成版面配置，呈現雜誌式新聞首頁的層級與節奏，不能只是新聞清單。
    5.可以參考 TIME.com 的設計精神，但不得複製其商標、圖片或原站內容。
    6.頁面必須支援桌面與手機版。
    7.可自行決定使用單一 HTML，或拆分為 index.html、styles.css 與 app.js。
    8.直接完成可開啟的網頁，不要另外產生冗長的分析或中間文件。
    ```

  - 利用這三個 Subagent 製作新聞網頁：
    ```md
    請使用 taiwan_news_researcher、time_style_researcher、news_page_builder 這幾個 Subagent 組成新聞網頁製作團隊，完成一個響應式臺灣新聞首頁。

    流程：
    1. taiwan_news_researcher 蒐集今天臺灣 10 則重要新聞。
    2. time_style_researcher 整理 TIME.com 首頁最重要的 5 個視覺規則。
    3. news_page_builder 整合前兩者的結果並完成網頁。
    ```
