## 6-2 在工具執行前後觸發 Hook

- 建立 PreToolUse Hook
  ```md
  想建立一個限定此專案的 PreToolUse Hook，想阻止 Codex 修改這個專案的內容：

  - 請先參考 Codex Hooks 官方文件，再依照正確格式在 .codex/hooks.json 建立：https://developers.openai.com/codex/hooks
  - 設定成只匹配 Codex 用來修改檔案的工具呼叫，例如 apply_patch
  - Hook 觸發後回傳退出代碼 2，並且要把阻擋原因寫到標準錯誤輸出
  - 阻擋原因請寫：這個專案目前不允許 Codex 直接修改檔案，請先取得使用者確認。
  - 不要執行 Python 或其他外部程式
  ```

## 6-3 避免提交機密訊息：UserPromptSubmit Hook

- 建立 UserPromptSubmit Hook
  ```md
  我想建立一個限定此專案的 UserPromptSubmit Hook，效果是防止重要資訊被提交給 Codex：

  - 請先參考 Codex Hooks 官方文件，再依照正確格式在 .codex/hooks.json 建立：https://developers.openai.com/codex/hooks
  - Hook 觸發時執行 check_sensitive_info.py 這個命令
  - 我是用 uv 管理專案，所以請使用 uv 執行
  - check_sensitive_info.py 稍後才會建立，這個步驟只需要完成 hook
  ```

- Hook 執行的命令：檢查提示詞的程式
  ```md
  我已經在這個專案建立好一個 UserPromptSubmit hook，但 check_sensitive_info.py 這個程式還未設計，請依照我的要求設計出這個程式：

  我希望它能在我把提示內容送給 Codex 前，從標準輸入接收 Codex 傳入的 JSON，從中先檢查內容有沒有夾帶敏感資訊(像 API Key、密碼、私鑰)，有的話就擋下來、不要送出。JSON 資料格式請參考 Codex Hooks 官方文件：https://developers.openai.com/codex/hooks

  程式需要：
  - 用 uv 執行，只用 Python 標準函式庫。
  - 根據官方文件取得標準輸入的 JSON 格式：
  - 從標準輸入讀取 Codex 傳來的 JSON，檢查裡面的 prompt 欄位。
  - 依 Codex Hooks 官方文件中提到的標準輸出，偵測到就擋下，沒問題就放行。
  - 命中時只說是哪種敏感資訊，不要把完整的機密值印出來。
  ```

## 6-4 傳送要求核准通知到 Telegram：PermissionRequest Hook

- 建立 PermissionRequest Hook
  ```md
  我想在這個專案中建立一個 PermissionRequest hook，效果是當 Codex 觸發核可批准事件時，會自動執行外部程式來處理通知。

  請先參考 Codex Hooks 官方文件，再依照正確格式在 .codex/hooks.json 建立：https://developers.openai.com/codex/hooks

  需求：
   - hook 觸發時執行 telegram_notification.py
   - 我是用 uv 管理專案，所以請使用 uv 執行命令
   - telegram_notification.py 稍後才會建立，這個步驟只需要完成 hook
  ```

- 取得 Telegram Bot CHAT_ID：
  ```md
  https://api.telegram.org/bot<你的API token>/getUpdates
  ```

- Hook 執行的命令：發送提醒通知到 Telegram
  ```md
  我已經在這個專案中建立好一個 PermissionRequest hook，但telegram_notification.py這個程式還未設計，請依照我的要求設計出這個程式：

  我希望這個程式能夠接收 Codex 的核可批准事件，將通知訊息轉發到我的 Telegram 機器人。程式會從表準輸入接收 Codex 傳入的 JSON，並根據內容產生 Telegram 通知訊息。JSON 資料格式請參考 Codex Hooks 官方文件：https://developers.openai.com/codex/hooks

  程式需要：
  1. 從 .env 檔案讀取 BOT_TOKEN 和 CHAT_ID
  2. 使用 Telegram Bot API 發送推播訊息
  3. 我是用 uv 工具來管理專案，所以請使用 uv 來操作
  4. 建立 .env 檔案來儲存 Telegram 設定
  5. 通知內容請包含 Codex 傳入的重點資訊，例如工具名稱、工作目錄、模型、輸入內容、Session / Turn 資訊
  6. 請處理可能的 UTF-8 等等的編碼問題，避免 Codex 傳入標準輸入格式，造成通知程式失敗或通知內容出現大量亂碼
  ```
