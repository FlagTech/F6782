```md!
下載成功後，回報影片標題與字幕檔儲存路徑；若無新影片或下載失敗，記錄原因並封存本次執行
每次執行皆為獨立任務，無需保留前次記憶
```

#### 使用 Codex 撰寫的提示內容手動建立排程

- 提示內容：

```md!
下載 YouTube 頻道 <YouTube 頻道網址> 過去 24 小時內發佈的最新一般影片字幕，排除 Shorts，並將提取出的字幕以 .txt 檔儲存在 caption_jp 資料夾中。 
執行規則： 
1. 以頻道的 /videos 頁面作為一般影片判斷來源，只處理一般影片，不處理 Shorts。 
2. 篩選影片上傳時間為執行時間往前 24 小時內的影片。 
3. 對每支符合條件的影片，開啟 YouTube 影片頁面，展開說明區，從「字幕記錄」點選「顯示轉錄稿」。 
4. 從顯示的轉錄稿面板擷取字幕內容，需包含時間戳。 
5. 字幕內容請採用面板中較完整的段落版本，避免把同一句話切成過細片段。 
6. 清除 YouTube 轉錄稿中非字幕內容，例如「1 分鐘 10 秒」、「5 分鐘」、「12 秒」這類插入的時間讀法；保留真正的時間戳格式，例如 [1:10]。 
7. 檔名格式固定為： 
YYYYMMDD_影片ID.txt 
例如： 
20260617_33tl6aoV188.txt 
8. 若同一影片已有舊檔，請覆蓋同名檔案，不要另外產生長檔名或重複檔。 
9. 任務完成後，確認 caption_jp 資料夾內每支影片只保留一個對應檔案。 
10. 完成後回報： 
- 成功下載的影片標題 
- 影片 URL 
- 字幕檔完整儲存路徑 
11. 若過去 24 小時內無符合條件的新一般影片，請回報「無新影片」並記錄原因。 
12. 若下載或擷取失敗，請回報失敗影片、原因，以及已完成/未完成狀態。 
13. 若無新影片或下載失敗，請封存本次執行。 
注意： 
- 不要下載 Shorts。 
- 不要使用字幕 API 或其他外部字幕下載工具取代頁面操作；字幕提取需從 YouTube 頁面中的「字幕記錄 → 顯示轉錄稿」取得。 
- 輸出的 txt 檔內容需為 UTF-8。 
- 不要保留前次執行的任務記憶；每次都重新檢查 YouTube 頻道。
```

#### 規劃模式實作翻譯字幕專案

- 提示內容：

```md!
建立 Python 程式，將 caption_jp 資料夾中 YouTube 影片的日文字幕（txt 格式）翻譯成繁體中文，並以相同檔名儲存到 caption_tw 資料夾中。
翻譯要求：
- 保留原文的段落結構
- 術語、人名、頻道名稱保留原文
- 翻譯風格自然流暢，符合台灣繁體中文習慣
```

### 從手機建立翻譯影片字幕的專案

#### 讓 Codex 幫我們寫 Skill

- 提示內容：

```md!
/Skill Creator
請整合以下排程需求及字幕翻譯功能，建立一個可供 Automation 使用的 Skill，讓它能每天自動檢查指定 YouTube 頻道是否有新影片；若有新影片，就自動取得原文字幕並執行 uv run python translate_captions.py 翻譯成繁體中文。由於此 Skill 將用於建立每日自動化排程，故需將完整流程整合至 Skill 中。 
請在專案資料夾的 .agents/skills/ 建立 SKILL.md，並設定以下 Frontmatter：

除了建立 SKILL.md 外，也請依目前專案架構補齊 Skill 所需的設定與說明，確保之後可直接透過 Automation 呼叫此 Skill。

排程需求如下：
[這裡貼上 7-1 節 Codex 生成的「提取新影片日文字幕」的提示內容]
```
