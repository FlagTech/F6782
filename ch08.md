# 第 8 章 使用 Cloud Task 接上 GitHub 工作流程

[本章範例](https://github.com/FlagTech/Page2Card)

## 8-1 將範例專案推送到 GitHub

### 下載範例專案

- 提示內容：

    ```
    幫我從以下網址下載範例專案到本機端：
    https://github.com/FlagTech/Page2Card.git
    ```

### 請 Codex 導覽專案結構

- 提示內容：

    ```
    不要修改任何檔案，幫我導覽這個專案的結構。先用一段話說明它的整體設計，再以檔案樹的格式列出主要的資料夾與檔案，並用白話一句話說明每個檔案的職責。
    ```

#### 專案啟動

- 提示內容：

    ```
    請幫我安裝並啟動這個專案。開始前，請先閱讀 README，確認專案的安裝與啟動方式。若過程中需要下載套件或執行需要我同意的指令，請先提出請求。遇到錯誤時，請先說明原因，再繼續處理。
    ```

- 若 Codex 啟動 server 後卡住，可改用背景方式啟動：

    ```
    幫我改以背景啟動的方式即可：
    uv run uvicorn page2card.main:app --host 127.0.0.1 --port 8000
    ```

#### GitHub 帳戶連接

- 提示內容：

    ```
    請幫我用 Scoop 安裝 gh 套件，並登入 GitHub
    ```

###  將專案推送到 GitHub 上

- 提示內容：

    ```
    請用 GitHub CLI 在我的帳號下建立一個名為 Page2Card 的儲存庫，將專案原本的遠端改成這個新儲存庫，然後把目前的程式碼推送上去。
    ```

## 8-2 建立 Issue 交給 Cloud Task 處理

### 透過 gh 建立 GitHub Issue

- 提示內容：

    ```
    請用 GitHub CLI 在本專案的儲存庫建立 Issue，標題為「為文章產生 AI 中文摘要，並生成可分享的圖卡」。內文請依「目標、背景、需求來源、實作範圍、不在範圍內、驗收條件」六個區塊撰寫，內容摘要自專案根目錄的 PAGE2CARD_ISSUE.md，並在「需求來源」註明完整規格見該檔，不要整份貼進 Issue。標籤請套用【標籤名稱使用中英文皆可】；負責人指定為【已在 GitHub 中存在的帳戶名稱】；里程碑請設定為【自定義階段目標名稱，中英文皆可】。完成後，把 Issue 的網址回報給我。如果遇到不能執行的步驟，寫出原因後，就繼續執行下一步。
    ```

###  建立雲端環境的事前準備

- 提示內容：

    ```
    請根據本專案的 README、PAGE2CARD_ISSUE.md 與目前 GitHub Issue，建立專案根目錄的 AGENTS.md。這份檔案會用於 Codex Cloud Environment，請整理 Codex 執行 Cloud Task 時需要知道的專案指引，包括安裝方式、常用指令、測試與 lint 指令、開發限制與驗收方式。內容請精簡明確，不要全文複製 README 或 Issue。
    ```

- 提交並推送 AGENTS.md：

    ```
    請幫我提交目前的變更，並推送到 GitHub。
    ```

### 連接 GitHub 建立雲端環境

- [Codex 網站](https://chatgpt.com/codex/)

- [OpenAI API Keys 申請頁面](https://platform.openai.com/api-keys)

### 透過 Cloud task 解 Issue 建立 PR

- 建立 PR（於本機對話視窗執行）：

    ```
    幫我把修改內容儲存成一個新的分支進行提交，並在 GitHub 上建立 PR。PR 的名稱與描述內容都要是繁體中文，標籤與里程碑都要與 Issue 相同。
    ```

## 8-3 用 Codex 審查 PR 並修正

### 請 Codex 幫忙審查 PR

- 讓 Codex review 變成中文：
  
  ```
  @codex review 請一律使用繁體中文回覆
  ```

###   呼叫 @codex幫忙修正 PR 問題

- 於 PR 審查區塊的 Reply 欄位輸入（先請它翻譯問題並提出修改方針）：

    ```
    @codex 請將以上審查意見翻成繁體中文，並提出修改計畫，先不要修改程式碼，等我確認後再執行。
    ```

- 確認方針後，請它依方針修改：

    ```
    @codex 請依照上述方針修改。
    ```
